# 线性表
---
## 概念
线性表是一种线性逻辑结构
是一种具有相同数据类型的n个数据元素的有限序列，其中n为表长，n=0时为空表
是一种前驱后继的关系
**和数组的区别**，有前驱和后继，使用数组的时候，你是不知道哪部分是你的线性表的(因为有**脏数据**)
解决的办法为:首先确定**起始位置**,然后只需要记录**表长**(size/length)
## 实现
就是简单的线性表，普通的不写了，写链式的
特别说明一下，有一种很有趣的初始化方法
![[数据结构1.png]]
使用链表实现
	结点：数据存放的地方(Node)

| 数据域(Data) | 指针域( * Next) |
| --------- | ------------ |
| 存放具体数据    | 下一个数据的地址     |
我们会发现，可以通过第一个指针知道第二个值，通过第二个找到第三个，那么如何找到第一个呢
答：写一个指针(头指针L)L的类型是本结构体,ta很特殊，1，通过他可以找到第一个Node，2，通过他可以找到整个链表

但是当我们使用这样的话就很麻烦，因为所有的代码都要考虑头指针这个特殊的情况
所以我们采用的是多个节点，Data域是什么无所谓，指针域指向第一个元素即可
这里具体的**增删改查**使用链表了哈，在链表里增删改查都是对结点来说的
初始化:
```c
......定义

typedef int Ntype;  
typedef struct Node{  
    Ntype data;  
    struct Node *next;  
}Node,*LinkList;
......函数

Node* Init_LinkList(){  
    Node* head = (Node*)malloc(sizeof(Node));  
    //为什么使用malloc，因为如果用别的，那么生成的是栈上的内存，函数执行完就没了
    head -> next = NULL;  
    return head;  
}
......主函数

int main(void){  
    LinkList L = Init_LinkList();  
        printf("%d",L->data);  
    return 0;  
}
```
增：
插入的时候有两种插入方法
- 头插法：每次插入的数据都放到头结点后，变成首元结点
	- 先申请空间，一个Node* s
	- 然后是不是就要开始链了，现在s是
![[数据结构2.png]]
我们需要让s插入的head和x之间，是不是得
先将s的指针域指向x
为什么呢
因为找到x的唯一途径是head
如果先让head指向s，那么x就没有任何指针能找到了
```c
//头插入  
LinkList Head_Insert(LinkList L,Ntype k){  
    Node* s = (Node*)malloc(sizeof(Node));  
    s -> data = k;//给数据域赋值  
    //**指针指向的本质就是指针赋值**  
    s -> next = L -> next;//所以这个本质是什么，是给s的next指向头的next，也就是，指向头结点下一个元素的位置  
    L -> next = s;//这个是头节点断开，去指向s  
    return L;//返回被修改后的L  
}  
int main(void){  
    LinkList L = Init_LinkList();  
    L = Head_Insert(L,10);  
    printf("%d",L->next->data);  
    return 0;  
}
```

- 尾插法：每次插入的数据都放到最后一个结点后面，称为尾结点
	- 尾结点难在，是不是我们只能知道头节点，但是找不到尾结点，是不是得寻找，得遍历
	- 由观察得——尾结点的指针域为NULL
	- 那么现在第二个问题就出现，是从头结点开始找呢，还是从首元结点开始呢？
	- 答：头结点，为什么，因为空链表，是不是根本就没有首元结点
```c
//尾插入  
LinkList Rear_Insert(LinkList L,Ntype k){  
    Node* s = (Node*)malloc(sizeof(Node));  
    s -> data = k;//给数据域赋值  
    //**指针指向的本质就是指针赋值**  
    s -> next = NULL;//尾插入最后是不是变成新的尾，尾的指针域为NULL  
    Node* p = L;//p在函数完成后释放，但是p->next因为p是Node*类型，所以并不是p这个局部变量的地址，而是L的地址，L本质又是Node的地址，所以这里可以使用  
    while(p->next!=NULL){  
        p=p->next;//p指向p后面的结点  
    }  
    p->next = s;//原尾结点指向s(新尾结点)  
    return L;//返回被修改后的L  
}  
int main(void){  
    LinkList L = Init_LinkList();  
    L = Head_Insert(L,10);  
    L = Rear_Insert(L,11);  
    printf("%d",L->next->next->data);  
    return 0;  
}
```
- 指定位置：在数据k后面插入数据k1
	- 相当于尾插法，但是找的并不是NULL，而是具体的值的位置
	- 然后插入的寻找不能从头结点开始寻找，为什么
	- 答：因为头结点的数据域是脏数据
	- 所以建议先写一个find(查找函数，见“查：”)
```c
LinkList Everything_Insert(LinkList L,int i,Ntype k){  
    Node* p=Find(L,i);  
    if (p==NULL){  
        printf("%d不存在",i);  
        return NULL;  
    }  
    Node* s = (Node*)malloc(sizeof(Node));  
    s->data = k;  
    s->next = p->next;  
    p->next = s;  
        return L;//返回被修改后的L  
}  
  
int main(void){  
    LinkList L = Init_LinkList();  
    L = Head_Insert(L,10);  
    L = Rear_Insert(L,11);  
    L = Everything_Insert(L,11,12);  
    printf("%d",L->next->next->next->data);  
    return 0;  
}
```
**无论是什么插入，第一步都是新建一个结点，将数据放到结点中**
**指针指向的本质就是指针赋值**
**malloc开辟的空间可以以指向该空间的指针命名**

## 优缺点
优点：
- 简单
- 支持随机存取
- 内存空间连续，资源利用率高
缺点：
- 插入和删除麻烦
适用场景为**静态数据**
