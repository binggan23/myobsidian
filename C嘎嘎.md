第3章 怎样使用类和对象 学习笔记
 
（全是类和对象的核心细节，好多容易踩的坑，整理一下方便后面翻）

最开始讲的是类对象的初始化，之前写代码总忘了初始化对象，导致数据乱飘，原来构造函数就是干这个的！
构造函数和类同名、没返回值，创建对象的时候自动调用，不用手动喊它。可以给它加参数，给不同对象初始化不同的值，也能重载，比如写一个无参的、一个带参数的，创建对象的时候它会自己匹配版本。还能用默认参数，少写好几个重载，挺省事的。
还有初始化列表！这个比在构造函数里给成员赋值高效多了，尤其是成员是别的类对象的时候，直接在初始化表里初始化，不用先默认构造再赋值。
析构函数就是构造的“反义词”，对象销毁的时候自动喊它出来做清理，比如释放new出来的内存。名字是 ~类名 ，也没返回值，而且不能重载，一个类只能有一个析构函数。
还有构造和析构的调用顺序，比如局部对象是“先构造的后析构”，别搞反了，不然析构的时候容易出问题。
讲完构造析构，就是对象数组和对象指针了。
对象数组就是一堆同类型对象凑成的数组，创建的时候每个元素都会调用构造函数，初始化可以用列表，比如 Student stu[3] = {Student(1), Student(2)};  剩下的元素会自动调用默认构造。
对象指针就是指向对象的指针，用法和普通指针差不多，用 -> 访问成员，比如 p->name ，和 (*p).name 是一个意思。
还有this指针！一开始真的懵，后来才懂，就是每个成员函数里自带的、指向“当前调用这个函数的对象”的指针。比如成员变量和函数里的局部变量重名的时候，用 this->age 就能区分开，不用硬改变量名。
接下来是const相关的一堆东西，真的绕到头疼，每次都要翻笔记：
- 常对象：用 const 修饰的对象，只能调用const成员函数，不能改它的成员变量。
- 常成员函数：函数后面加个 const ，相当于跟编译器保证“我这个函数绝不修改对象的成员”，const对象只能喊这种函数干活，普通对象也能用。
- 常数据成员：类里的const成员，必须在初始化列表里初始化，不能在构造函数里赋值，而且一旦定了就改不了。
- 指向对象的常指针：指针本身不能改，只能钉死指向这个对象，但可以通过它修改对象里的值。
- 指向常对象的指针：不能通过这个指针修改对象的值，哪怕对象本身不是const的也不行。
- 对象的常引用：引用本身不能改，也不能通过引用修改对象，用来当函数参数超安全，既避免了拷贝，又不会不小心改了原对象。
（瓦达西注意到，要明确是指针本身不能变，还是指向的对象不能变）
然后是对象的动态建立和释放，就是 new 和 delete 嘛。用 new 创建对象的时候会自动喊构造函数， delete 的时候喊析构函数，注意数组的话要用 delete[] ，不然会内存泄漏！
还有对象的赋值和复制：
- 赋值是两个已经存在的对象之间用 = 赋值，默认的浅拷贝直接把成员的值复制过去，但如果有指针成员的话，会导致两个对象指向同一块内存，析构的时候重复释放，所以这种情况要自己写拷贝构造函数实现深拷贝。
- 拷贝构造是创建新对象的时候，用一个已有的对象来初始化，格式是 类名(const 类名 &obj) ，必须是引用，不然会无限递归调用，直接炸了。
不同对象间要共享数据的话，就用静态成员！
静态数据成员是属于整个类的，所有对象共享同一份，要在类外初始化，比如 int Student::count = 0; ，不用创建对象，直接用类名就能访问。
静态成员函数也属于类，没有this指针，只能访问静态数据成员，不能直接碰非静态成员——毕竟它不知道你要访问哪个对象的成员。
最后是类模板，就是把类的类型参数化，写一个通用的类，比如 template <typename T> class Stack ，这样就能支持int、char、自定义类各种类型了，不用每个类型都重写一遍类，省事很多。
